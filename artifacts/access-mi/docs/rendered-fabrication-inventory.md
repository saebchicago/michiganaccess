## Method

The scanner walks `artifacts/access-mi/src/pages` and `artifacts/access-mi/src/components`,
excluding tests, the legitimate snapshot-metrics pipeline (`src/utils/snapshotMetrics.ts`,
already CI-gated), and generated `src/data/county-*.ts` files. Two patterns are detected:

- **A. Module-level stat arrays** - `const X = [...]` declarations whose elements are object
  literals with at least two numeric properties. If the array body contains no `source:` field,
  the array is flagged `likely-hardcoded-data` with disposition `SOURCE-IT`. If it does contain
  a `source:` field, it is flagged `likely-sourced` / `FALSE-POSITIVE` (kept in the report for
  reviewer audit, but the structural marker is present).
- **B. JSX text-node numeric literals** - percentages, comma-separated counts, per-N rates, and
  dollar-magnitude tokens appearing between `>` and `<` in JSX. These are flagged `needs-review`
  with disposition `NEEDS-REVIEW`; reviewer judgment is required because some are valid copy
  (state averages cited inline), some derive from sourced upstream data via interpolation that
  the scanner cannot statically prove, and some are genuine fabricated literals.

Disposition vocabulary (for the SOURCE-IT and NEEDS-REVIEW rows the owner will work through):

- `SOURCE-IT` - replace the hardcoded literal with a sourced value (pipeline import, generated
  county data, or `DataProvenance`-attributed component) and add a primary-source label.
- `REMOVE` - delete the rendered surface entirely (e.g., demo placeholders).
- `FALSE-POSITIVE` - heuristic noise; e.g., chart-axis tick array, config map, the literal is
  already sourced via comment or imported manifest.
- `NEEDS-REVIEW` - owner inspection required before classifying.

## Known limitations

- **Heuristic, not a parser.** Does not build an AST. May miss numeric literals built via
  `.toFixed()`, runtime concatenation, or other dynamic string assembly.
- **JSX text-node match is narrow.** Only catches numeric tokens between literal `>` and `<`
  in source. Misses literals embedded in `{...}` interpolation. This is intentional - it
  trades recall for precision and avoids false-positive carpet-bombing chart-tick code.
- **Stat-array threshold is conservative.** Requires at least two object literals with at
  least two numeric-property hits. Single-element configs and small lookup tables are not
  flagged.
- **No semantic understanding.** A chart-tick array like `[{x:0,y:5}, {x:1,y:7}]` will be
  flagged as `likely-hardcoded-data` even when the surface is decorative. Reviewer marks
  these `FALSE-POSITIVE`.
- **`source:` field check is structural only.** An array that declares `source:` is treated
  as sourced even if the cited source does not actually back the figures - the structural
  affordance is present, audit of the citation itself is out of scope here.
- **Excludes the legitimate pipeline.** `src/utils/snapshotMetrics.ts` and generated
  `src/data/county-*.ts` files are explicitly skipped because they are already gated by
  `scripts/check-fabrication.mjs`.
